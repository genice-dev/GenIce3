# リリース前チェックリスト（GenIce3）

このチェックリストは、リリース候補を確定して公開するまでの最終確認用です。

## 1. スコープと版管理

- [ ] リリース対象コミット（またはタグ候補）を確定した
- [ ] 以降に取り込む変更の条件（バグ修正のみ等）を決めた
- [ ] バージョン表記を統一した（`pyproject.toml` / `README.md` / `RELEASE_NOTE.md`）
- [ ] 既存タグとの整合を確認した（重複・命名ミスなし）

## 2. ドキュメント

- [ ] `RELEASE_NOTE.md` が最新変更を反映している
- [ ] `CHANGES.md` と `RELEASE_NOTE.md` の内容が矛盾していない
- [ ] 破壊的変更（互換性注意点）が明記されている
- [ ] 新機能・主要修正の使用例が最低1つある
- [ ] **ドキュメント本文が英語のみで記述されている（原則）**
- [ ] 英語以外の記述が必要な場合は、例外理由を明記した

## 3. 機能互換性（要注意）

- [ ] 旧スクリプトからの移行で問題になりやすいオプション変更点を確認した
- [ ] `pol_loop_1` / `pol_loop_2` など分極関連の移行説明を確認した
- [ ] CIF 入力時の挙動（H 位置あり/なし）を期待どおりに確認した
- [ ] `partial_order` 廃止の影響を確認した
- [ ] `ice21` と alias エントリの解決が期待どおりに動作する

## 4. テストと動作確認

- [ ] テストスイートを実行し、失敗がない
- [ ] `tests/identity` を実行し、比較結果を確認した
- [ ] 代表的な CLI 実行例をクリーン環境で再現できた
- [ ] 代表的な YAML 設定例を実行し、出力を確認した
- [ ] WebAPI 関連の最小動作を確認した（対象リリースに含む場合）

## 5. 配布物品質

- [ ] sdist / wheel をビルドし、必要ファイルが含まれている
- [ ] エントリポイント（CLI / plugin group）が有効
- [ ] 依存関係の上限・下限で致命的不整合がない
- [ ] 既知の制限事項を文書化した

## 6. 公開準備

- [ ] 公開手順（タグ作成、配布、告知）を手順化した
- [ ] ロールバック/ホットフィックス方針を決めた
- [ ] 公開後に確認する監視項目を定義した
- [ ] 最終レビュー担当者と承認フローを確定した
