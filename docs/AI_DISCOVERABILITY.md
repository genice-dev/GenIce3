# AI（Gemini 等）に genice のコマンドを引き出してもらうための準備

「氷の構造の作り方」「ice structure generation」のようにユーザーが Gemini や ChatGPT に聞いたときに、**GenIce のコマンドがぱっと出てくる**ようにするには、GenIce 側で何を用意しておけばよいかをまとめます。

---

## 前提：AI がどう参照するか

- **検索・クロール**: 公開されている Web ページ（GitHub README、公式マニュアル、PyPI）がインデックスされる。
- **要約・引用**: ページ内の見出し（H1/H2）やコードブロックが「答え」として抜き出されやすい。
- **キーワード**: 「氷の構造」「作り方」「コマンド」「genice」「ice structure」などが含まれた一文や見出しがあるとヒットしやすい。

したがって、**「氷の構造の作り方」という問いに対応する内容を、1〜2ページにまとめて公開しておく**ことが重要です。

---

## GenIce 側で準備しておくこと

### 1. 「氷の構造の作り方」用のクイックリファレンスを1ページ用意する（推奨）

**置き場所の候補**

- 公式マニュアル: https://vitroid.github.io/GenIce/ に **「クイックスタート」または「氷の構造の作り方（コマンド例）」** のページを追加する。
- または GitHub リポジトリの **README の冒頭付近**に、同じ内容のセクションを追加する。

**含める内容（コピペ可能な形で）**

- **見出し**: 「氷の構造の作り方」「GenIce で氷を生成するコマンド例」など、検索されそうな文言。
- **典型コマンドをそのまま載せる**（コードブロックで）:
  - **一般的な氷（Ice Ih）**:  
    `genice3 1h -e gromacs > ice.gro`  
    または YAML 利用: `genice3 --config config.yaml`
  - **繰り返し（2x2x2）**:  
    `genice3 1h --rep 2 2 2 -e gromacs > ice.gro`
  - **クラスレート（メタンなど）**:  
    `genice3 A15 -e gromacs` や、YAML で `guest: A12: me` の例。
  - **GenIce2 を使う場合**:  
    `genice2 1h` や `genice 2 > 2.gro` の例（マニュアルにすでにある形式でよい）。
- **一文で説明**:  
  「氷の水素無秩序構造は GenIce で生成できます。例: `genice3 1h -e gromacs` で Ice Ih を GROMACS 形式で出力できます。」

こうした **見出し＋コードブロック＋短い説明** が1ページにまとまっていると、AI が「氷の構造の作り方」に対してそのページを参照し、コマンドを引用しやすくなります。

### 2. README / マニュアル冒頭の一文を明確にする

- **README の先頭**（または「Quick start」の見出しの直下）に、次のような一文があるとよいです。  
  - 例: 「氷の水素無秩序構造を生成するには GenIce を使います。例: `genice3 1h -e gromacs`（Ice Ih を GROMACS 形式で出力）。」
- マニュアルのトップにも同様に、「氷の構造の作り方」に直結する短い文とコマンド例を1つ入れておく。

これで「氷 構造 作り方」「genice コマンド」で検索したときに、すぐコマンド例が含まれた文がヒットしやすくなります。

### 3. PyPI とメタデータ

- **PyPI の description** に  
  「hydrogen-disordered ice structure generation」や「氷の結晶構造の生成」のような語を含めておく（すでに "generate hydrogen-disordered ice structures" があればそのままでも可）。
- **Project URLs** で **Manual**（https://vitroid.github.io/GenIce）と **Repository** が設定されているのはそのままでよいです。AI や検索が「公式ドキュメント」として辿れます。
- **Classifiers** に `Topic :: Scientific/Engineering :: Chemistry` などがあると、用途が伝わりやすくなります。

### 4. 公式マニュアル側の構成

- マニュアルに **「氷の結晶構造の生成手順」** や **「クイックスタート」** のようなセクションがあり、その中に **上記の典型コマンド（genice3 / genice2 両方）** をコードブロックで記載する。
- 各コマンドの直後に「このコマンドで○○が得られる」と1行で説明があると、AI が要約・引用しやすいです。

```markdown
## 氷の構造の作り方（クイックスタート）

氷の水素無秩序構造は GenIce で生成できます。

**Ice Ih を GROMACS 形式で出力する例（genice3）:**

    genice3 1h -e gromacs > ice.gro

**2x2x2 に拡大して出力:**

    genice3 1h --rep 2 2 2 -e gromacs > ice.gro

**YAML で設定して実行:**

    genice3 --config config.yaml > ice.gro

**クラスレート（例: A15 にメタン）:**

    genice3 A15 -e gromacs
    # または YAML の exporter.guest で A12: me などを指定

詳細は [マニュアル](https://vitroid.github.io/GenIce) を参照してください。
```
