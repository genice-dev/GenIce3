### 設計上の検討事項

- [ ] セミクラスレートハイドレートなど複雑な構造が欲しいユーザーが想定されるが、GenIce3 にセミクラスレート用オプションは未実装
- [ ] ケージ位置とケージを構成する酸素の情報を API から渡さないと、GUI 上でゲストの場所を指定できない
- [ ] ユーザーは GUI でイオンをドープしたら即座に画面に表示されることを期待する
- [ ] API に再生成させると待たされ、意図した結果にならない場合もある
- [ ] API と GUI を緊密に連携させると設計が非常に難しくなる

### 提案: 一方向ワークフロー

- [ ] API は初期構造（座標＋ケージ情報）だけを提供する
- [ ] GenIce GUI はそれ以降を単独で結晶を装飾し、最後にオプション候補を提示する
- [ ] 一方向のデータフローにすることで責務を明確にする
- [ ] ただし GenIce で実装済みの機能を Svelte 側で相当量再構築する必要が生じる可能性
- [ ] ハイブリッド案: 候補位置だけ GenIce が返し、配置は GUI で行う。最終検証・最適化のみ API に任せる

### GenIce 本体に必要な拡張（GUI 再開時に検討）
